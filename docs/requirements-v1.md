# 智能终端助手 需求分析文档 v1

## 1. 产品概述

### 1.1 产品定位

智能终端助手（termhelper）是一个运行在 Linux 终端中的智能代理工具。用户以自然语言描述操作意图（如 `? 删除所有std开头的文件夹`），软件调用大语言模型（LLM）将自然语言转换为可执行的 Shell 命令，辅以命令说明和安全警告，最终帮助用户安全、高效地完成终端操作。

### 1.2 目标用户

- 对 Linux 命令行不够熟悉的用户，希望用自然语言驱动终端操作
- 需要快速完成常见运维任务但不想记忆命令参数的开发者
- 希望降低误操作风险的终端使用者

### 1.3 核心价值

| 痛点 | 解决方案 |
|------|----------|
| 记不住命令和参数 | 自然语言输入，LLM 翻译为命令 |
| 不确定命令是否安全 | 自动风险评级，橙色警告危险操作 |
| 看不懂命令含义 | 自动生成命令分解说明 |
| 复杂交互式操作（如 sudo apt upgrade）需手动介入 | PTY 接管模式，LLM 代替用户与程序交互 |
| 重复调查系统环境 | 环境调查结果持久化，跨会话复用 |

## 2. 功能需求

### 2.1 FR-01 自然语言输入

**触发方式：** 用户在终端中输入 `? <自然语言描述>`，例如 `? 请删除所有std开头的文件夹`。

**输入处理：**
- 软件捕获 `?` 前缀后的全部文本作为自然语言查询
- 将查询发送至 LLM 进行意图理解

### 2.2 FR-02 命令生成

**功能描述：** LLM 将自然语言查询转换为可执行的 Shell 命令。

**输入：** 用户的自然语言查询 + 已存储的系统环境上下文（如有）

**输出：**
- 一个或多个候选 Shell 命令
- 每个命令的自然语言说明（命令用途、关键参数解释）

**验收标准：**
- 生成的命令语法正确，可在目标 Shell 中执行
- 说明文字通俗易懂，覆盖命令用途及关键参数

### 2.3 FR-03 环境调查（Agent 模式）

**功能描述：** 当 LLM 无法确定用户系统环境时，可发起环境调查——向用户申请执行诊断命令，获取系统信息后再生成最终命令。

**典型场景：**
```
用户: ? 更新系统
LLM:  需要了解系统包管理器，申请执行: cat /etc/os-release
用户: 同意
LLM:  (获取到 Debian 信息) 生成命令: sudo apt update && sudo apt upgrade
      储存结论: "用户正在使用 Debian Linux"
```

**调查流程：**
1. LLM 判断当前查询需要环境信息
2. LLM 生成诊断命令并请求用户授权
3. 用户同意后，执行诊断命令并获取输出
4. LLM 基于诊断结果生成最终命令
5. 环境调查结果持久化存储，后续会话可载入复用

**验收标准：**
- LLM 可主动发起环境调查请求
- 诊断命令需经用户授权方可执行
- 调查结果正确存储并可被后续会话加载

### 2.4 FR-04 命令说明生成

**功能描述：** 为生成的每一条命令提供人类可读的说明。

**说明内容要求：**
- 命令整体用途概述（一句话）
- 各关键参数/子命令的作用解释（一句话）
- 颜色编码：正常信息默认色，危险警告用橙色

**示例：**
```
命令: rm -rf std*
说明:
  rm   — 删除文件或目录
  -r   — 递归删除目录及其内容
  -f   — 强制删除，不提示确认
  ⚠ 该命令具有危险性，删除的文件无法恢复 [橙色]
```

### 2.5 FR-05 安全分析

**功能描述：** 自动分析命令的风险等级并给予相应提示。

**风险等级定义：**

| 等级 | 触发条件 | 提示方式 |
|------|----------|----------|
| 安全 | 纯读取操作（ls, cat, echo 等） | 无额外警告 |
| 注意 | 文件写入/修改操作 | 温和提示 |
| 危险 | 不可逆删除（rm -rf）、格式化（mkfs）、磁盘操作（dd） | 橙色醒目警告 |
| 提权 | 需要 sudo/root 权限 | 警告需管理员权限，提示接管模式 |

**验收标准：**
- 至少识别删除类、格式化类、提权类三类危险命令
- 危险警告以橙色渲染
- 危险命令在执行前需额外确认

### 2.6 FR-06 用户交互选项

**功能描述：** 命令生成后，向用户展示交互选项。

**选项菜单：**

| 选项 | 行为 |
|------|------|
| **运行** | 在子 Shell 中执行命令 |
| **复制** | 将命令复制到系统剪贴板，由用户自行粘贴运行 |
| **修改** | 输入进一步的修改指示，LLM 重新生成命令 |

**验收标准：**
- 三项选项均可用
- 选项选择方式便捷（数字/字母快捷键）

### 2.7 FR-07 命令执行

**功能描述：** 在用户确认后执行命令。

**模式一：普通执行**
- 适用场景：非交互式命令
- 实现方式：spawn 一个子 Shell 进程执行命令
- 执行完成后展示退出码和输出

**模式二：PTY 接管执行**
- 适用场景：需要交互的命令（如 sudo apt upgrade 需要确认、输入密码等）
- 实现方式：spawn 一个 PTY（伪终端），LLM 监控程序输出并自动进行交互操作
- 触发条件：
  - 命令包含 `sudo` 等需要交互的组件
  - 用户选择「运行并接管」
- 行为：LLM 充当中间人，读取程序输出，自动发送应答输入

**重要说明：** 普通用户无法直接向终端模拟打字（不能通过向 stdin 写入来模拟交互），因此交互式命令必须通过 PTY 模式执行。

**验收标准：**
- 普通命令可正常 spawn 执行
- PTY 模式下 LLM 能正确识别程序提示并自动应答
- 执行过程可被用户中断

### 2.8 FR-08 环境上下文管理

**功能描述：** 存储和加载系统环境调查结果，避免重复调查。

**存储内容：**
- 操作系统及发行版信息
- 包管理器类型
- 已安装的关键工具及版本
- 其他 LLM 认为有复用价值的调查结论

**存储方式：** 本地文件持久化（如 JSON/YAML 配置文件）

**生命周期：**
- 首次使用：环境上下文为空，LLM 按需调查
- 后续使用：自动加载已有上下文，LLM 判断是否仍有效
- 更新：系统环境变化时 LLM 可发起重新调查

**验收标准：**
- 环境上下文可持久化存储
- 新会话启动时自动加载已有上下文
- 上下文变化时可更新

### 2.9 FR-09 LLM 对话管理

**功能描述：** 管理多轮对话上下文，支持命令修改和追问。

**支持场景：**
- 用户对生成的命令不满意，输入修改指示
- LLM 需要澄清用户意图时发起提问
- 环境调查时为诊断命令请求授权

**上下文管理：**
- 每个会话维护对话历史
- 对话历史随每次交互追加
- 单个会话内 LLM 可引用之前的所有交互

**验收标准：**
- 支持多轮对话修改命令
- 对话历史在会话内保持连贯

## 3. 非功能需求

### 3.1 NFR-01 性能

| 指标 | 要求 | 备注 |
|------|------|------|
| 冷启动时间 | ≤ 2s | 从用户输入 `?` 到显示交互界面 |
| LLM 响应延迟 | 取决于 API | 需有加载状态提示 |
| 环境上下文加载 | ≤ 100ms | 文件读取 |

### 3.2 NFR-02 可靠性

- LLM 调用失败时给出友好错误提示，不崩溃
- 命令执行超时时允许用户中断
- 网络异常时提供重试机制

### 3.3 NFR-03 安全性

- 危险命令必须经用户二次确认
- 诊断命令必须经用户授权方可执行
- 不在用户不知情的情况下执行任何 Shell 命令
- 环境上下文文件中不存储敏感信息（如密码、Token）
- LLM API Key 安全存储（环境变量或配置文件，不硬编码）

### 3.4 NFR-04 可用性

- 命令说明使用用户可理解的自然语言，避免术语堆砌
- 颜色编码清晰一致：危险=橙色，正常=默认色，成功=绿色
- 错误信息包含可操作的解决建议

### 3.5 NFR-05 可扩展性

- LLM 后端可替换（支持不同模型提供商）
- 命令安全规则可扩展（新增危险模式识别）
- 支持新增交互选项类型

## 4. 系统架构概要

### 4.1 模块划分

```
┌──────────────────────────────────────────────┐
│                 CLI 入口层                     │
│  捕获 ? 前缀输入 → 路由到核心引擎              │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│                核心引擎                        │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐  │
│  │ 对话管理  │ │ 上下文管理│ │ 安全分析器   │  │
│  └──────────┘ └──────────┘ └──────────────┘  │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐  │
│  │ LLM客户端 │ │ 命令执行器│ │ 结果显示/交互│  │
│  └──────────┘ └──────────┘ └──────────────┘  │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│              基础设施层                        │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐  │
│  │ 剪贴板   │ │ 文件存储  │ │ Shell/PTY    │  │
│  └──────────┘ └──────────┘ └──────────────┘  │
└──────────────────────────────────────────────┘
```

### 4.2 核心数据流

```
用户输入 ?查询
    │
    ▼
CLI 入口 ──→ 加载环境上下文 ──→ 构造 LLM 请求
                                        │
                                        ▼
                                   LLM API 调用
                                        │
                    ┌───────────────────┼───────────────────┐
                    ▼                   ▼                   ▼
              需要环境调查？        生成命令+说明        无法理解？
                    │                   │                   │
                    ▼                   ▼                   ▼
           申请执行诊断命令      安全分析+渲染显示    请求用户澄清
                    │                   │
                    ▼                   ▼
           用户授权→执行→回传    交互选项菜单（运行/复制/修改）
                    │                   │
                    ▼           ┌───────┼───────┐
           新一轮 LLM 调用       ▼       ▼       ▼
                            运行命令   复制   修改指示
                               │       │       │
                        ┌──────┼──┐   剪贴板  新一轮
                        ▼         ▼           LLM 调用
                  普通spawn   PTY接管
```

### 4.3 技术栈

| 层 | 技术选型 | 说明 |
|-----|---------|------|
| 编程语言 | 仓颉（Cangjie） 1.1.0 | 项目约束 |
| 构建工具 | cjpm | 仓颉包管理器 |
| LLM 集成 | HTTP Client（stdx） | 调用 LLM API |
| 终端渲染 | 标准输出 + ANSI 转义序列 | 颜色编码 |
| 配置存储 | JSON 文件 | 环境上下文、LLM 配置 |
| 剪贴板 | xclip / wl-clipboard | Linux 剪贴板工具 |

## 5. 用户故事

### US-01 新手删除文件
> 作为不熟悉 Linux 命令的用户，我希望输入"删除所有 tmp 开头的文件"，系统能告诉我该用什么命令、解释命令含义、并警告风险，让我安全地完成操作。

### US-02 开发者更新系统
> 作为开发者，我希望输入"更新系统"后，系统能自动识别我的 Linux 发行版和包管理器，生成正确的更新命令，并在需要 sudo 时帮我自动处理交互。

### US-03 不满意初版命令
> 作为用户，当 LLM 生成的命令不符合预期时，我希望能够输入修改指示让它重新生成，而不是退出手动重写整个查询。

### US-04 只想复制命令
> 作为有经验的用户，我不需要软件帮我执行命令，只想快速拿到命令复制到剪贴板，自己决定何时执行。

## 6. 术语表

| 术语 | 定义 |
|------|------|
| 自然语言查询 | 用户在 `?` 后输入的日常语言描述，如"删除所有 std 开头的文件夹" |
| 诊断命令 | LLM 为调查系统环境而请求执行的探测性命令，如 `cat /etc/os-release` |
| 环境上下文 | 经调查获得的系统环境信息集合，持久化存储供后续复用 |
| PTY 接管 | 通过伪终端让 LLM 监控程序输出并自动应答，模拟人工交互 |
| 普通执行 | spawn 子进程执行命令，适用于不需要交互的命令 |
| 安全分析 | 对生成命令进行风险评级并产生对应警告提示的过程 |

## 7. 待澄清问题

1. **LLM 提供商选型：** 使用哪个大模型 API？（OpenAI / Anthropic / 本地模型）API 配置方式如何？
2. **多 Shell 支持：** 是否需要支持 bash 以外的 Shell（zsh, fish）？生成的命令是否需适配不同 Shell 语法？
3. **会话历史：** 是否需要保存跨启动会话的对话历史，还是仅保留环境上下文？
4. **命令执行工作目录：** 命令是在用户当前终端所在目录执行，还是在软件内部维护工作目录？
5. **并发安全：** 是否允许多个终端窗口同时使用本软件？若允许，环境上下文读写如何同步？
6. **剪贴板降级策略：** xclip/wl-clipboard 均不可用时，是否降级为仅显示命令让用户手动复制？
7. **命令执行超时：** 是否需要超时限制？默认超时时间是多少？
8. **国际化：** 是否需要支持中英文以外的语言？LLM 返回的命令说明语言如何控制？
